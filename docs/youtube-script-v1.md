## [인트로 — 10초]

안녕하세요, 김성진입니다.
오늘은 YOLO26 모델로 산업용 결함 검출 파이프라인을 처음부터 끝까지 만든 과정을 보여드리겠습니다.

---

## [문제 정의 — 30초]

철강 제조 공정에서는 표면 결함을 사람이 눈으로 검사합니다.
크리스, 주름, 롤 자국 같은 결함이 10가지나 되는데, 검사원마다 판정이 다르고, 놓치는 것도 많습니다.

이 프로젝트에서는 GC10-DET이라는 실제 철강 결함 데이터셋을 사용했습니다.
10개 클래스, 약 2,300장의 이미지로, 클래스 간 불균형이 12배 이상인 까다로운 데이터셋입니다.

---

## [파이프라인 개요 — 20초]

전체 파이프라인은 5단계입니다.

첫째, 데이터 분석 — EDA로 클래스 분포와 결함 크기를 파악합니다.
둘째, 전처리 — CLAHE로 대비를 강화합니다.
셋째, 학습 — YOLO26 Nano 모델을 파인튜닝합니다.
넷째, 개선 — 데이터 증강으로 불균형을 해소합니다.
다섯째, 배포 — ONNX로 변환해서 C# WPF 앱으로 실시간 추론합니다.

---

## [학습 결과 — 40초]

Baseline 모델의 mAP50은 0.635였습니다.
이미지 해상도를 데이터셋에 맞게 조정하자 0.701로, 10% 이상 올랐습니다.

여기서 모델을 키우면 어떨까 해서 Small 모델도 학습했는데, 0.720으로 2.7% 상승.
파라미터가 4배인데 성능 차이는 작습니다.

그래서 데이터 쪽을 개선했습니다.
Albumentations로 소수 클래스를 집중 증강해서 불균형을 12배에서 2.4배로 줄였더니,
Nano 모델이 0.726으로 — Small을 넘어섰습니다.

핵심 교훈: 작은 데이터셋에서는 모델을 키우는 것보다 데이터를 개선하는 게 더 효과적입니다.

---

## [ONNX 배포 — 30초]

학습된 모델을 ONNX로 변환하면, PyTorch 없이 추론할 수 있습니다.
C#과 ONNX Runtime으로 만든 InspectView 앱에서 바로 사용합니다.

RTX 3060 기준, GPU 추론 11.5밀리초 — 87 FPS.
산업 현장의 라인 스캔 속도를 충분히 커버합니다.
