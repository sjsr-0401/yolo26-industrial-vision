1.5 이미지 해상도 분포

핵심 인사이트: 각 데이터셋의 원본 해상도가 크게 다르다(200px ~ 2048px). 따라서 학습 시 imgsz 파라미터를 데이터셋별로 다르게 설정해야 한다.
  • NEU-DET (200x200): imgsz=640 — 원본보다 큰 해상도는 노이즈만 증폭
  • DeepPCB (640x640): imgsz=640 — 원본 해상도와 동일
  • GC10-DET (~2048x1000): imgsz=1024 — 원본 정보를 최대한 보존

2. 전처리 (Preprocessing)

2.1 CLAHE (Contrast Limited Adaptive Histogram Equalization)

논문: Zuiderveld, K. (1994). "Contrast Limited Adaptive Histogram Equalization." Graphics Gems IV, Elsevier. pp. 474-485.

왜 CLAHE를 선택했는가?

  • 철강 표면 결함은 대비(contrast)가 낮아 육안으로도 구분이 어려운 경우가 많다
  • 일반 히스토그램 평활화(HE)는 전체 이미지에 균일하게 적용되어 이미 밝은 영역을 과도하게 밝게 만드는 문제가 있다
  • CLAHE는 이미지를 타일(tile)로 나누어 지역적(local) 대비 강화를 수행하고, clipLimit으로 과도한 증폭을 제한한다
  • 산업용 비전 검사에서 사실상 표준 전처리로 사용되고 있다
# CLAHE 적용 (LAB 색공간, L 채널에만 적용) lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB) clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8)) lab[:, :, 0] = clahe.apply(lab[:, :, 0]) img_enhanced = cv2.cvtColor(lab, cv2.COLOR_LAB2BGR)

위 비교에서 볼 수 있듯이, CLAHE 적용 후 히스토그램이 더 균일하게 분포되며, 결함 영역의 대비가 향상된다. 특히 GC10-DET의 어두운 영역에서 효과가 두드러진다.

2.2 데이터 증강 (Data Augmentation)

Mosaic Augmentation

논문: Bochkovskiy, A. et al. (2020). "YOLOv4: Optimal Speed and Accuracy of Object Detection." arXiv:2004.10934. 인용 15,000+

4개의 훈련 이미지를 하나로 합성하여 배치 크기를 효과적으로 증가시킨다. 소형 객체가 다양한 컨텍스트에서 학습될 수 있어, 소형 객체 검출 성능 향상에 기여한다. 본 프로젝트에서 mosaic=1.0으로 전체 에포크에 적용하되, 마지막 15 에포크에서는 비활성화하여(close_mosaic=15) 모델이 실제 이미지 분포에 적응하도록 했다.

Copy-Paste Augmentation

논문: Ghiasi, G. et al. (2021). "Simple Copy-Paste is a Strong Data Augmentation Method for Instance Segmentation." CVPR 2021. Google Brain. 인용 1,500+

다른 이미지에서 객체 인스턴스를 복사하여 현재 이미지에 붙여넣는다. 소수 클래스의 인스턴스를 증가시켜 클래스 불균형을 완화한다. GC10-DET(불균형 12.4x)에는 copy_paste=0.2를 적용했으나, NEU-DET(200x200, 작은 이미지)에는 복사-붙여넣기가 비효과적이어서 copy_paste=0으로 설정했다.

CutMix

논문: Yun, S. et al. (2019). "CutMix: Regularization Strategy to Train Strong Classifiers with Localizable Features." ICCV 2019. 인용 5,000+

이미지의 일부 영역을 다른 이미지로 교체하고, 레이블도 면적 비율에 따라 혼합한다. Mixup(Zhang et al., ICLR 2018)의 발전형으로, 공간적 정보를 보존하면서 정규화 효과를 제공한다. 본 프로젝트에서 mixup=0.1로 적용했다.

2.3 클래스 불균형 대응

Focal Loss

논문: Lin, T.-Y. et al. (2017). "Focal Loss for Dense Object Detection." ICCV 2017. Facebook AI Research. 인용 25,000+

표준 Cross-Entropy Loss에 modulating factor (1-p_t)^gamma를 곱하여, 쉽게 분류되는 샘플(easy negatives)의 loss 기여를 줄이고, 어려운 샘플(hard examples)에 집중하도록 한다. GC10-DET처럼 클래스 불균형이 심한 데이터셋에서 특히 효과적이다. YOLO26은 내부적으로 Focal Loss 변형을 사용한다.

# Focal Loss (개념) FL(p_t) = -alpha_t * (1 - p_t)^gamma * log(p_t) # gamma=0이면 일반 CE Loss, gamma=2가 일반적 (Lin et al.)

3. 학습 (Training)

3.1 모델 선택: YOLO26

YOLO26(2026)은 anchor-free, NMS-free end-to-end 검출 모델이다. 기존 YOLO 시리즈(v5~v11)와의 핵심 차이점:

  • NMS-free: 후처리 단계에서 Non-Maximum Suppression이 불필요 (DETR, Carion et al., ECCV 2020의 영향)
  • Anchor-free: Anchor box 사전 정의 불필요 (FCOS, Tian et al., ICCV 2019의 영향)
  • DFL 제거: Distribution Focal Loss를 제거하여 경량화
  • ONNX 배포 시 43% 빠름 (CPU 기준, ONNX Runtime end-to-end)

3.3 v2 실패 → v3 성공 분석

v2 실패 사례 (교훈)
  • NEU-DET: imgsz=1280으로 설정 → 200x200 원본을 6.4배 upscale → 노이즈 증폭, mAP50 0.730→0.658 하락
  • GC10-DET: batch=auto → OOM 크래시, 7 에포크 만에 종료

교훈: "더 크게, 더 많이"가 항상 좋은 것은 아니다. 데이터셋의 특성(원본 해상도, 크기)이 하이퍼파라미터 선택을 결정해야 한다.

v3 개선 전략
  • NEU-DET: 원본 해상도에 맞춰 imgsz=640 유지, 과도한 증강 제거
  • GC10-DET: batch=4 고정 (OOM 방지), imgsz=1024로 정보 보존
  • 증강 강도 완화: scale=0.7(0.9에서), degrees=5(10에서)

4. 후처리 (Post-processing)

4.1 NMS-free End-to-End Detection

논문: Carion, N. et al. (2020). "End-to-End Object Detection with Transformers (DETR)." ECCV 2020. Meta AI. 인용 8,000+
관련: Tian, Z. et al. (2019). "FCOS: Fully Convolutional One-Stage Object Detection." ICCV 2019. 인용 3,500+

왜 NMS-free가 중요한가?

기존 YOLO(v5~v8)는 추론 후 NMS(Non-Maximum Suppression)를 수행해야 한다. NMS는 겹치는 박스를 제거하는 휴리스틱 후처리로, 여러 문제가 있다:

  • IoU 임계값에 민감: 너무 높으면 중복 검출, 너무 낮으면 인접 객체 삭제
  • 배포 복잡성: ONNX 모델 출력이 raw boxes → NMS를 C#/C++에서 직접 구현해야 함
  • 속도 저하: GPU에서 CPU로 데이터 이동 필요 (bottleneck)

YOLO26은 학습 단계에서 이미 중복 검출을 억제하도록 훈련되므로, ONNX 모델의 출력이 바로 최종 검출 결과이다. 이는 DETR(Carion et al., ECCV 2020)이 제안한 end-to-end 패러다임을 CNN 기반 검출기에 적용한 것이다.

# YOLOv5 배포 (NMS 필요) raw_output = onnx_session.run(input) # [N, 8400, 85] boxes, scores = parse_raw_output(raw_output) indices = custom_nms(boxes, scores, iou_threshold=0.45) # 직접 구현! final_detections = boxes[indices] # YOLO26 배포 (NMS-free) final_detections = onnx_session.run(input) # 바로 최종 결과!

4.2 Confidence Calibration

논문: Guo, C. et al. (2017). "On Calibration of Modern Neural Networks." ICML 2017. 인용 6,000+

딥러닝 모델의 confidence score는 실제 정확도와 일치하지 않는 경우가 많다 (overconfident). Temperature Scaling은 출력 logit을 temperature 파라미터 T로 나누어 신뢰도를 보정하는 간단하면서도 효과적인 방법이다. 산업 현장에서는 confidence threshold에 따른 Pass/Fail 판정이 중요하므로, calibration은 실용적 가치가 높다.

4.3 ONNX Runtime 배포

학습된 모델을 ONNX(Open Neural Network Exchange) 형식으로 변환하여 C# WPF 앱에서 OnnxRuntime으로 추론한다. YOLO26의 NMS-free 특성 덕분에 ONNX 출력을 바로 사용할 수 있어 배포 코드가 단순해진다.

5. 결론 및 교훈

5.1 핵심 성과

  • GC10-DET mAP50: 0.635 → 0.701 (+10.4% 개선) — 데이터셋 맞춤형 튜닝
  • DeepPCB mAP50: 0.984 — Nano 모델로도 near-perfect 달성
  • 3개 산업 도메인(철강 2종 + PCB) 커버
  • NMS-free ONNX 배포 — 후처리 코드 불필요

5.2 엔지니어링 교훈

  1. 데이터셋 이해가 먼저다: EDA 없이 하이퍼파라미터를 정하면 v2처럼 실패한다
  2. 원본 해상도가 imgsz를 결정한다: 200px 원본을 1280으로 upscale하면 역효과
  3. 클래스 불균형은 정확도의 적이다: GC10-DET의 12.4x 불균형이 성능 ceiling의 원인
  4. Nano vs Small 트레이드오프: 정확도가 필요하면 Small, 속도가 우선이면 Nano
  5. 실패를 기록하라: v2 실패 → 원인 분석 → v3 성공, 이 과정 자체가 엔지니어링 역량
